# Milestone 8.11 — Partial-spend notes with change outputs

## Phase 1: AIR refactor + diag green — SHIPPED

This milestone makes shielded spends *partial*: a spend can produce a
new shielded note (a "change output") that the spender retains, instead
of forcing every spend to consume the entire leaf value.

**Cryptographic change:**
Before: `unshield_amount + fee == v_in` (full spend, no change).
After:  `unshield_amount + fee + v_out == v_in` (partial spend, v_out
in a new shielded note H(sk_out, r_out, v_out) that the chain appends
to the STARK pool).

The chain sees only the hash of the new note (`output_leaf`). The
spender retains the (sk_out, r_out, v_out) secrets to spend it later.

## Phase boundaries

Per Option A discipline, M8.11 is split into four phases with explicit
checkpoints. **Only Phase 1 has shipped.**

| Phase | Scope | Status |
|-------|-------|--------|
| **1** | AIR refactor + diag green | **DONE** |
| 2     | Soundness adversarials against change outputs | not started |
| 3     | qstark_py bindings + QChain chain integration | not started |
| 4     | Network adversarial integration | not started |

Phase 1 alone does NOT make M8.11 usable from QChain. Phase 1 proves
the cryptographic core is consistent.

## What Phase 1 changed

### Trace layout (qstark/src/hash_air/m86_air.rs + m86_native.rs)

- Width grew **84 → 151 columns**
  - Cols 84-86: output witness `(sk_out, r_out, v_out)`
  - Cols 87-150: 64-bit decomposition of v_out
- Active rows grew **144 → 152**
  - New 8-row block at rows 144-151 for the output leaf hash
- Trace length stays **256** (already padded; one more block fits)

### Constants added

```
COL_SK_OUT, COL_R_OUT, COL_V_OUT             // output witness cols
COL_BIT_OUT_START, COL_BIT_OUT_END           // v_out bit decomp
OUTPUT_LEAF_BLOCK                            // new block index
OUTPUT_BOUNDARY_ROW                          // transition INTO output block
OUTPUT_LEAF_LAST_ROW                         // M86_ACTIVE_ROWS - 1 = 151
NULLIFIER_LAST_ROW                           // FIXED: was M86_ACTIVE_ROWS-1
                                             // (which used to be the nullifier
                                             // row; now it's the output_leaf row)
```

### Constraints (113 → 194)

| Group | Count | Change |
|-------|------:|--------|
| 1 — Hash round | 12 | unchanged |
| 2 — Merge boundary | 12 | gating extended to exclude output boundary too |
| 3 — Nullifier boundary | 12 | unchanged |
| **3.5 — Output-leaf boundary** | **12** | **NEW: mirrors Group 3 but reads sk_out/r_out/v_out** |
| 4 — Dir binary | 1 | unchanged |
| 5 — Within-block static | 5 | unchanged |
| **6 — Global witness static** | **6** | **was 3; now also pins sk_out/r_out/v_out** |
| 7 — Leaf-block binding | 3 | unchanged (output block uses Group 3.5 instead) |
| **8 — Bit binary** | **128** | **was 64; now also v_out bits** |
| **9 — Decomposition** | **2** | **was 1; now also v_out decomposition at row 0** |
| **10 — Value conservation** | **1** | **NEW: v_in − v_out − (unshield+fee) == 0 at row 0** |

The old M8.8-A1 amount-sum boundary assertion (state[6][0] == unshield+fee)
is REMOVED. It's replaced by Group 10 transition constraint, which is more
expressive — it can reference cell-minus-cell.

### Boundary assertions (13 → 16)

| Assertion | Status |
|-----------|--------|
| Capacity at row 0 (4 cells) | unchanged |
| Root at ROOT_ROW (4 cells) | unchanged |
| Nullifier at NULLIFIER_LAST_ROW (4 cells) | unchanged (row constant updated) |

### New periodic column

`is_output_boundary`: fires at OUTPUT_BOUNDARY_ROW (143 at depth 16), zero
elsewhere. Drives Group 3.5 and gates Group 2 (merge boundary) so it
doesn't fire at the new transition.

## Honest scope notes — what's NOT verified

Phase 1 does **not** prove M8.11 is sound. It proves the constraint
system is consistent on an honest witness. Real soundness verification
needs Phase 2.

### Specifically NOT done in Phase 1

- **Winterfell prove+verify against the new AIR**. Diag manually
  evaluates each constraint group; Winterfell composes them via a
  combination polynomial. A degree-spec mismatch (e.g., a constraint
  that's actually degree N when registered as N-1) wouldn't be caught
  by diag. Phase 2 runs the real prover and verifier.
- **Soundness adversarials for change-output forgery**. The natural
  attack class includes:
  - Spender picks v_out such that v_in − v_out − unshield − fee ≡ 0
    (mod p) but v_out's u64 representation overflows
  - Spender claims output_leaf = X but the trace actually produces
    output_leaf = Y
  - Spender uses inconsistent witness columns (v_out = 5 in one place,
    v_out = 10 in another)
  - Range proof on v_out is faked (b_out[i] outside {0, 1})
  Phase 2 writes ~14 adversarial tests, one per attack class.
- **qstark_py bindings**. The PyO3 wrapper in `qstark_py/src/lib.rs`
  still has the old `M86Inputs` shape. Building the wheel against the
  new AIR will fail to compile until Phase 3 updates it.
- **QChain Python integration**. `STARKAnonTransaction` doesn't carry
  an output_leaf field, the chain doesn't append the new leaf when
  mining the spend, and `create_stark_anon_tx` doesn't accept partial
  amounts. All Phase 3 work.
- **Network propagation**. New tests for malicious peers that tamper
  output_leaf, v_out, etc. between hops. Phase 4 work.
- **Other tests in qstark/tests/ are untested in Phase 1**. The
  M86Witness construction sites in m86_smoke, m86_soundness, m86_bench,
  m86_gap_a_soundness were updated to include `v_out=0` (full-spend
  behavior) but Phase 1 does NOT verify these tests still pass. They
  may have additional issues (e.g., they could still be running
  Winterfell, which Phase 1 hasn't verified). Phase 2 will run them.

## Why Phase 1 ships in this state

Diag green is the cheapest possible proof that the AIR refactor
preserved structural consistency. It catches:
- Constraint groups that fail to satisfy on honest witnesses (off-by-one
  in row gating, wrong cell references, missing constraint variants)
- New constraint groups that don't fire when expected (the M8.11
  output-leaf boundary)
- Removed constraints that left orphan assertions (the old amount-sum)

It does NOT catch:
- Constraint degree mismatches (Winterfell-level)
- Soundness failures (attacker can produce a fake-but-valid trace)
- Integration bugs (chain doesn't actually use the new shape)

These are real concerns. They're Phase 2/3 concerns. Phase 1 is the
minimum scoped checkpoint that proves "the AIR refactor didn't break
the math."
